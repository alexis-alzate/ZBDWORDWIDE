# Documentación del Diseño Responsive

Este documento detalla las decisiones de diseño y arquitectura que se tomaron para hacer que el sitio de 2BD Boutique sea completamente responsive y mantenible.

## Filosofía General

El enfoque principal fue pasar de un layout "manual y rígido" a uno "automático y flexible". En lugar de posicionar elementos con coordenadas fijas, se creó un sistema basado en reglas de flujo (Flexbox) que permite que los componentes se adapten de forma natural a cualquier tamaño de pantalla.

Las claves de este sistema son:
- **Flexbox:** Para repartir el espacio y controlar el alineamiento (`flex-1`, `justify-between`, `items-center`).
- **Breakpoints de Tailwind (`sm:`, `md:`, `lg:`):** Utilizados en todas las clases relevantes (tipografía, padding, márgenes) para adaptar el diseño a diferentes anchos de pantalla.
- **Variables de Control ("Perillas de Ajuste"):** Constantes al inicio de cada componente que guardan strings de clases de Tailwind. Esto permite controlar las dimensiones y estilos desde un solo lugar, facilitando el mantenimiento y la experimentación sin tocar el JSX.

## Análisis por Componente

### 1. Layout General (`App.jsx`)

- **Problema Anterior:** En contenidos cortos, el footer flotaba a mitad de página.
- **Solución:** Se implementó un layout de "sticky footer".
  - `min-h-screen` en el contenedor principal asegura que la app ocupe, como mínimo, toda la altura de la pantalla.
  - `flex flex-col justify-between` convierte el contenedor en una columna que "empuja" al `Header` hacia arriba y al `Footer` hacia abajo.
  - El elemento `<main>` usa `flex-1`, lo que hace que ocupe todo el espacio sobrante en el medio, y con `items-center justify-center` alinea `CollectionsSection` perfectamente en el centro de ese espacio.

### Layout Esquema (Diagrama de Flujo)

```
BackgroundCarousel (w-full, min-h-screen)
│
└── overlay flex-col (min-h-screen, text-white)
    │
    ├─ Header (mobile: columna centrada / desktop: fila)
    │   ├─ H1: "2BD WORLDWIDE"
    │   └─ Logo central (normal en mobile, absolute en md+)
    │
    ├─ main flex-1 (centra contenido)
    │   └─ CollectionsSection
    │       ├─ Título "Colecciones" (clases desde data.js)
    │       └─ Logos en fila con wrap + uno destacado
    │
    └─ Footer (eslogan + textos, bloque único)
```
*Así es como fluyen los componentes dentro del carrusel: Header arriba, sección centrada, footer abajo, todo dentro de un `flex flex-col` que reparte la altura.*

### 2. `BackgroundCarousel.jsx`

- **Estado:** Un único estado `index` controla el set de imágenes activo. `useEffect` con `setInterval` se encarga de la rotación y la limpieza para evitar fugas de memoria.
- **Responsive:** El contenedor de cada set de imágenes usa `flex flex-col md:flex-row`. Esto hace que las imágenes se apilen en vertical en móviles y se pongan en fila en pantallas más grandes.
- **Transiciones:** La duración de la transición de opacidad viene de `CAROUSEL_CONFIG.duracionTransicion` en `data.js` y se aplica vía `style`, permitiendo un control centralizado.

### 3. `Header.jsx`

- **Layout Dinámico ("Transformer"):**
  - **Móvil:** Es una columna centrada (`flex flex-col items-center`). El logo está en el flujo normal del documento para evitar solapamientos.
  - **Desktop (`md:` y superior):** Se transforma en una fila (`md:flex-row`). El logo vuelve a su `md:absolute` para lograr la composición de diseño original, ya que ahora hay espacio suficiente.
- **Tamaños Adaptativos:** Tanto el logo como el título escalan su tamaño y espaciado (`tracking`) en los diferentes breakpoints para mantener la armonía visual.

### 4. `CollectionsSection.jsx`

- **Variables de Control:** Todas las decisiones de layout (márgenes, tamaños, gaps) están en constantes al inicio del componente. Esto permite modificar la jerarquía visual rápidamente sin tocar JSX.
- **Estilos desde `data.js`:** El título de la sección importa sus clases de estilo (`opacidad`, `borde`, `fondo`) desde `TEXTOS.colecciones.estilos`, permitiendo experimentar con su look and feel de forma centralizada.

### 5. `CollectionCircle.jsx`

- **Prop `destacado` (booleano):** Actúa como un interruptor. Si es `true`, aplica un set de clases de tamaño más grandes (`logoSizeDestacado`). Si es `false`, usa las clases base (`logoSizeBase`). Esto permite diferenciar un elemento de la lista (el logo central) de forma elegante y sin duplicar código.

### 6. `Footer.jsx`

- **Bloque Único:** Se movió el eslogan "MÁS QUE UNA MARCA" para que forme parte del footer. Al compartir estilos de texto y tener un espaciado (`gap-1`) mínimo, se percibe como un bloque de texto único y coherente, dando un cierre más sólido a la página.

### 7. `data.js` (El "Cerebro")

- Sigue siendo la fuente de verdad para el **contenido** (`TEXTOS`, `LOGOS`).
- Ahora también centraliza la **configuración** (`CAROUSEL_CONFIG`).
- Y empieza a centralizar decisiones de **estilo** (`TEXTOS.colecciones.estilos`), actuando como un embrión de "Design System" o sistema de "tokens".
